# Module 2: First Workflow

## Overview
In this module, you will build and run your very first GitHub Actions workflow. The goal is to understand how the file is structured and exactly where it needs to live in order for GitHub to detect it.

**Official Reference**: [GitHub Actions Quickstart Guide](https://docs.github.com/actions/quickstart)
https://docs.github.com/en/actions/reference/workflows-and-actions/contexts#github-context

---

## The `.github/workflows` Directory
For GitHub to discover and execute any GitHub Actions workflows in your repository, you **must** save your YAML files in a specific directory at the root of your project:
`👉 .github/workflows/`

If a workflow YAML file is placed anywhere else, GitHub will ignore it.

---

## Your First Pipeline

We are going to create a simple pipeline that outputs system information about the trigger event, the user, and the runner (the virtual machine executing our jobs).

### Key Concepts Introduced
- **`name`**: Gives your workflow a friendly name in the GitHub UI.
- **`run-name`**: (Optional) Customizes the name of individual workflow runs. In this example, it dynamically includes the name of the user who triggered it.
- **`github` context**: Contexts are a way to access information about workflow runs, environments, jobs, and steps. We use context variables like `${{ github.actor }}` to print out the username dynamically.
- **Array Syntax in `on:`**: We can trigger the workflow on multiple events using an array directly: `on: [push, workflow_dispatch]`

---

## The Workflow Code

Below is the YAML representation of our first pipeline. You can also find this code in the standalone `first-pipeline.yml` file in this folder.

```yaml
# .github/workflows/first-pipeline.yml

name: GitHub Actions Quickstart
run-name: ${{ github.actor }} is testing their first pipeline! 🎉

on: [push, workflow_dispatch]

jobs:
  Explore-GitHub-Actions:
    runs-on: ubuntu-latest
    
    steps:
      - name: Information about the trigger
        run: echo "🎉 The job was automatically triggered by a ${{ github.event_name }} event."
        
      - name: Information about the OS
        run: echo "🐧 This job is now running on a ${{ runner.os }} server hosted by GitHub!"
        
      - name: Information about the branch
        run: echo "🔎 The name of your branch is ${{ github.ref }} and your repository is ${{ github.repository }}."
        
      - name: Check out repository code
        uses: actions/checkout@v4
        
      - name: Confirm successful checkout
        run: echo "💡 The ${{ github.repository }} repository has been cloned to the runner."
        
      - name: List contents of the repository
        run: |
          echo "🖥️ The workflow is now ready to test your code on the runner."
          ls -la
        
      - name: Final Status Updates
        run: echo "🍏 This job's status is ${{ job.status }}."
```

## How to Execute It

1. **Ensure the Target Directory Exists:**
   Make sure the `.github/workflows/` directory exists at the root of your git repository.
   
2. **Move the File to the Correct Location:**
   Copy the `first-pipeline.yml` file from the current folder and place it directly into `<your-repo-root>/.github/workflows/first-pipeline.yml`.

3. **Commit and Push:**
   Commit the new file to your repository and push the changes.
   ```bash
   git add .github/workflows/first-pipeline.yml
   git commit -m "Create my first GitHub Actions pipeline"
   git push origin main
   ```

4. **Watch it Run in the UI:**
   - Go to your repository on GitHub.com.
   - Click on the **"Actions"** tab at the top.
   - Look for the workflow run titled: `[Your-Username] is testing their first pipeline! 🎉`
   - Click into the run, and select the **Explore-GitHub-Actions** job on the left.
   - Expand the individual steps and review the output generated by the variables we injected!
